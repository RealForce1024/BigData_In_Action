19. 网络连接（重点 NAT）

NAT方式 这里一定要使用NAT方式(与外网隔离，配置不随外网环境变化改变，同时可以访问外网)
桥接 指向网关 ，虚拟机与物理机的地位是同等的。但随网络环境的变化而随之改变。
NAT 内网与外网的隔离 地址是隔离 ，两个不同的网段(宿主机与虚拟机通过vmware虚拟的路由进行连接的，可以配置vmware vmnet8对应的网关)，但是可以访问外网。

  1. Io总线（推荐即可）
  2. 磁盘类型 (SATA 笔记本 SCSI服务器(更好)) VM虚拟出来的
  3. 磁盘是用新的 （1）还是现存的 不要使用第三种（可能会删掉数据）
  4. 盘符给出的大小
    1. 20G 开始是不会都占用的,他是上限
    2. 系统多大就是多大
    3. 不要 allocate all disk space now （建立完之后就会将磁盘全部分配）
  5. 磁盘文件 （不用改）

    网络连接配置原理 (物理机方式)

    这里看下 NAT虚拟网络配置
    多个物理主机服务器 通过连接 路由器（功能最强）或交换机或网桥 工作在一起
    物理硬件上连接了，但还是需要一些配置。
    硬件 除了网线连接之外 （需要有网卡，是可以连接多个网络）
    软件 设置 操作系统 centos6 配置网卡的信息 tcp 协议的信息
    - ip 3个 配ip由网关决定的 交换机或路由器都是网关，他们自己都有ip 例如 192.168.33.1
    2-254 十进制 8位 实际上是二进制 11111111 255
    255是广播地址 整个网络

    vmware 会虚拟出来一个路由网关，这里可以通过vmware软件配置修改的。
    约定俗称一般 用..*. 1吧 网关地址 192.168.22.1 （我个人的设置，以后就按照这个来）

    vmnet8 则是虚拟出来的网卡

    - netmask 3  子网掩码 是用来判断自己属于哪个网段的 255.255.255.0  与 ip 下  1与任何数都是原来的数字 ， 0与2 是0   所以就是 192.168.33.0
    网段的计算： 将ip地址的2进制&子网掩码的二进制 = 网段地址的192.168.33.2 & 255.255.255.0 = 192.168.33.0
    - gateway  3 网关  网络数据的总出口
    - dns   3  域名解析服务器  解析域名 映射ip  先找本地 hosts windows/system32/drivers/etc/hosts 文件 主机名 ip地址 
    从本地hosts寻找域名 => ip映射信息。 (如果本地hosts没有该域名，则想外部的dns服务器查询)
    找到ip地址 就发出真是的请求
    上外网就写 网关地址即可 。 可以通过网关找域名地址 8.8.8.8 ，建议直接填写网关 192.168.33.1 网关地址
    - hwaddr 网卡自带的 硬件地址 数据链路层 第二层
    - 集群很大的时候，主机名比ip肯定好记录... ，但是主机名不能直接访问 ping server  -=>  unknown host  要么公网注册域名....败家》。。主机太多。 内部使用即可。 配置 **/etc/hosts**   内部都需要配置
     > 192.168.33.2 server01
     192.168.33.3 server02
     192.168.33.4 server03
     每台机器都需要配置，否则 网络不可达

    配置网卡

    vi /etc/sysconfig/network-scaripts/ifcfg-eth0

    DEVICE=eth0 #网卡1
    TYPE=Ethernet #网络类型
    ONBOOT=yes #是否开机启动
    BOOTPROTO=static #ip地址设置为静态的
    IPADDR=192.168.22.201 #ip地址 (看虚拟网关的配置，要符合其网段及范围)
    NETMASK=255.255.255.0
    GATEWAY = 192.168.22.1 #虚拟网关的配置
    DNS1=192.168.23.1 #宿主机的网关地址

    虚拟机网络配置原理

    逻辑上是分开的
    首先联网
    联网方式 Nat虚拟网络 方式 与上面的方式连接原理是一样的。
    但是没有
    真是的物理网卡 教室、寝室的交换机或路由器(忽略，这是不同的网络)
    虚拟网卡 vmnet8 （虚拟的网络）

    虚机有虚拟出的网卡 eth0 ifconfig查看 （当成是真网卡）

    路由器 wmware 提供 会虚拟产生出来，看不到 转发 地址转换 充当网关

    配置软件 ip
    首先看网关 ip地址 在 wmware软件的配置中 edit -> virtual network editor
    有三个方式可以选择，我们选择选择 vmnet8 nat

    子网网段 有默认的 subnet Ip 可以修改
    nat settings -> gateway ip 配置 192.168.22.1
    IPADDR: 192.168.22.201
    BOOTPROTO = STATIC
    ONBOOT = YES
    GATEWAY: 192.168.21.1
    NETMASK:255.255.255.0
    DNS1: 192.168.23.1 默认配置成宿主网关地址即可。

    windows 的配置也是和虚拟机的配置是一样的。
    这样是可以访问外网的。

    另外另种方式

    • 桥接模式
      • 虚机与物理机 共用交换机或路由器
      • 物理网卡 虚拟网卡(安装虚机的时候 vmnet0 auto bridge) 她会连接那块桥接的虚拟网卡 wmnet0(在windows上) 本质上是个网桥。 与路由器交换机是一样的
      • 集线器 hub 不做地址转发，与物理网卡共用同一个网段
      • windows内部请求都要通过网桥 –> 然后网卡–>外网
      • 只需要配置虚拟机的ip地址即可。
      • 192.168.21.x 这里教室间大家没有隔离。 但是回家后 静态地址需要改。
    • host only 不建议用
      • 虚机访问不了外网
      • 虚拟机都连接在vmnet1网卡上 wmnet1就作为网关，虚拟机配置ip的时候就根据wmnet1地址来配置
      • 缺点上不了外网
