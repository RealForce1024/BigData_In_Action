方法定义

  1. 可以省略返回值类型，scala自动判断返回值类型
    def addOne(m: Int): Int = m + 1 // 注意 参数列表 后 :returnType。
    def m1(x:Int ,y : Int) : Int = x*y
    简写：def m1(x:Int,y:Int) = x*y

  2. 参数类型签名不能少
    def m1(x,y)=x*y //该行代码shell中执行错误，idea中编译不通过

scala> def m2(x,y)=x*y
<console>:1: error: ':' expected but ','found.//说明语法定义需要:paramType

  1. 特殊情况，无参数列表的时候可以省略()
    def m22 = 1+2 <==def m22() = 1+2 //如果没有参数可以直接省略 参数列表签名，否则需要齐全。返回值类型除特殊情况（如递归）外可以可有可无
    m22 <== m22()
    4 //result

  2. 特殊情况 递归 出口处需要些返回类型

函数定义

匿名函数（无 def 关键字 无methodName 通过=> 指向函数体）
（x:Int,y:Int）=> x+y
注意：匿名函数不要返回值类型

scala> (x:Int):Int=>x+1 //匿名函数 参数类型如同方法一样都需要指定，但是返回值类型是不能指定的。
<console>:1: error: not a legal formal parameter

因此总结： 参数签名要写完整，方法或函数的返回值类型尽量不写。
在方法体中如果有多行代码，可以使用{ } 格式化代码,如果写在一行，需要使用 ;隔开

scala> (x:Int,y:Int)=> x+y
res5: (Int, Int) => Int = <function2>

scala> (x:Int,y:Int,c:Double)=> x+y
res6: (Int, Int, Double) => Int = <function3>

function2 function3 …. 重载….

小结 

方法： def 名称:(参数列表):[Type] = 方法体
      def m1(id:Int,name:String) = {...}
函数： 参数列表 => 方法体
      val func = (x:Int,y:Int)=>x+y

Higher-order function
In mathematics and computer science, a higher-order function (also functional, functional form or functor) is a function that does at least one of the following:

  • takes one or more functions as arguments (i.e., procedural parameters),
  • returns a function as its result.[disputed – discuss]

All other functions are first-order functions. In mathematics higher-order functions are also known as operators or functionals. The differential operator in calculus is a common example, since it maps a function to its derivative, also a function.

In the untyped lambda calculus, all functions are higher-order; in a typed lambda calculus, from which most functional programming languages are derived, higher-order functions that take one function as argument are values with types of the form (\tau_1\to\tau_2)\to\tau_3.
